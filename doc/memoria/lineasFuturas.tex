	Tras la elaboración de la memoria de este proyecto y la implementación del mismo, se advierten varios ámbitos donde hay alguna carencia tecnológica o se observa una oportunidad de desarrollo futuro.
	
	Una necesidad de este sistema es una migración hacia TDD. Un sistema tan importante como una votación, que debe minimizar los errores al máximo por la criticidad de los mismos, debería apoyarse en un sistema automatizado de tests. Deberían implementarse tests unitarios y punto a punto para todos los subsistemas y funcionalidades del sistema. Con esto, se puede asegurar la integridad del mismo y facilitar los cambios en el software, provocando un software más estable.
