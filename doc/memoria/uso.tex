Una vez configurado el programa, e inicializado el tracker, en este capítulo se indicará cómo se debe proceder a levantar el servicio. También se realiza alguna pequeña aclaración sobre los distintos manuales que se deben utilizar.
\subsection{Levantar el servicio}\label{levantar}
\par
Para arrancar el tracker y que el sistema entre en funcionamiento, se debe levantar el servicio. 
\par
Esta operación se deberá realizar también en caso de que la máquina caiga, o tras cualquier operación de mantenimiento del sistema. Es decir, cada vez que ocurra un evento por el cual debamos volver a poner en funcionamiento el sistema.
\par
Para levantar el servicio, deberemos trabajar como usuario normal, y no como superusuario (como hemos hecho en la instalación y en la configuración). Para ello, se deben revisar los permisos en las carpetas involucradas, y comprobar que sean los pertinentes.
\par
Se ejecutarán en la consola las siguientes instrucciones, de acuerdo a los casos expuestos en la instalación y la configuración:
\begin{center}
\verb|export PATH=$PATH:/opt/roundup/bin|
\end{center}
\begin{center}
\verb|roundup-server pfc=/opt/roundup/trackers/pfc|
\end{center}
\par
Por comodidad en el uso, se recomienda encarecidamente el uso del script ''levantar.sh'' que se adjunta en el CD, y contiene símplemente esas dos instrucciones. Para utilizar este script, habría que alojarlo dentro del directorio ''home/pfc''. Para levantar la aplicación bastará unicamente con situarnos en ese directorio desde la consola y ejecutar:
\begin{center}
\verb|./levantar.sh|
\end{center}
\par
\subsection{Manual de usuario}
\par
Para un correcto uso del sistema, se ven involucrados los siguientes documentos:
\begin{itemize}
	\item Documentación de uso de Round-up Issue Tracking (incluida en el CD adjunto, y también disponible y actualizada en \cite{roundupweb}).
	\item Cualquier manual, normativa o documentación propuesta para la utilización de este sistema, comunicación en los roles más bajos, etc, así como cualquier otro documento que sea considerado pertinente por las Fuerzas Armadas.
\end{itemize}
\par
Aunque el funcionamiento del sistema es muy intuitivo y sencillo, cualquier duda podrá ser aclarada en la documentación del programa, por lo que esa documentación debe estar disponible. \par
Igualmente, el programa no deberá entrar en producción hasta que los usuarios finales tengan pleno conocimiento de las normativas de uso impuestas por las Fuerzas Armadas.
